# 小程序开发进度及功能总结 — 2026-04-25

本文档总结了目前已实现的微信小程序端核心功能及技术要点。

## 1. 核心业务流程
- **账号绑定与登录**：支持微信一键登录。首次登录的学生需输入老师提供的 **6位唯一学生码**（如：AB1234）完成账号与微信 OpenID 的绑定（已支持在登录页主动解绑学生码）。
- **课程列表**：登录后展现所有已加入班级的聚合课程列表（支持多对多班级体系），采用响应式“Bouncy Cards”设计，支持分色主题、完成度统计，并在完成所有句子的练习后根据得星比例实时显示奖杯等级（金、银、铜）。
- **跟读练习（Adventure）**：核心跟读流程已跑通，支持分段配图背景切换、录音、回放及提交。

## 2. 已实现的特色功能
### 2.1 智能录音系统
- **录音稳定性优化**：针对微信 `touchend` 事件在 DOM 变化时可能丢失的问题，实现了“透明点击层（Hit Area）”技术，确保录音停止极其可靠。
- **实时进度反馈**：每完成一句跟读，系统会自动触发 **流星动画（Flying Star）**，并在顶部进度条实时同步，增加学习趣味性。
- **本地进度缓存**：支持本地存储跟读记录，用户退出页面或网络断开时，已录好的句子不会丢失，再次进入可无缝衔接。

### 2.2 播放与反馈
- **双模播放**：
  - **听范读**：支持从课程主音频（Master Audio）中精确跳转（Seek）到对应句子的时间点播放。已修复首次播放跳回起点的时间戳问题，提供更精准的跟读体验。
  - **听自己**：点击红色小喇叭按钮，可回放当前句子的用户本地录音，方便对比纠音。
- **自动跳转**：录音结束后，系统会自动滚动并激活下一句，实现流畅的跟读体验。

### 2.3 激励与活跃（打卡系统）
- **每日打卡弹窗**：每日首次进入小程序时，自动弹出打卡奖励卡片。
- **连胜统计（Streak）**：显示用户练习的连胜天数及累计完成总句数。

### 2.4 内容管理优化
- **分离式素材上传**：支持在无音频材料的情况下单独添加课程句子内容，增强了教案编辑的灵活性。

## 3. 技术架构要点
- **后端通信**：全量使用自研 `request` 工具类，支持 JWT 鉴权、Token 自动刷新及后端 API 交互。
- **云开发集成**：使用微信云开发（CloudBase）的 `uploadFile` 接口直传录音文件到腾讯云 COS，避开传统 HTTPS 上传的域名限制和性能损耗。
- **组件化开发**：打卡系统已封装为独立组件 `checkin`，便于在不同页面复用。

## 4. 待确认/后续计划
